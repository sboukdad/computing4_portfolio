## Description
Explain what the project is supposed to do:

This program is an introduction to SFML. This is a library that allows us to
 create programs or applications provided the interface for graphics, audio,
 window management, etc. This program produces a window with a white circle 
 and a bowl of ramen in the middle. This ramen (sprite) can move via keyboard
 up, down, left, right keys. There is also a soothing audio that goes with 
 the image featured in this example. 

### Features
Describe what your major decisions were and why you did things that way:

Ramen felt like an easy image that would exist in many pixelated forms, so I
 went with that. Alongside that, the decision to make my added feature audio
 tied in and I thought it would set the mood for my image really well. 
 Another decision I made in this program was the desire to make the circle and 
 sprite resize with the window - allowing the image to stay the same regardless
 of size of window.

### Issues
I ran into an issue where I was unable to compile the cpp file if I did not
 include the full path to my files, but realized I could write the relative 
 path and just make sure I am in the correct folder in terminal when I go 
 to recompile the program. 

A recent issue I ran into when I made my first submission attempt was the fact
 that the autograder was not able to properly compile and identify sfml-app. 
 I had been using a previous version of SFML and was able to remove it and 
 update it, and was successfully able to link all the necessary files and 
 passed through the autograder   


### Extra Feature
The special features I incorperated into this project include making the sprite 
 move using the keyboard, was a fun audio in the background as the program runs!
